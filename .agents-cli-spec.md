# Agent Spec: Personalized Newsletter Agent (ADK-based)

## Overview
A personalized newsletter generation system built using the Google Agent Development Kit (ADK). The agent consumes a user's curriculum vitae (CV)/resume to establish their professional profile, skill set, and areas of interest. It then crawls and searches daily news headlines and article databases to compile, filter, and draft a summary of today's major happenings that are directly relevant and beneficial to the user.

To establish cross-session memory, the agent utilizes ADK's **Memory Bank** to remember user preferences, feedback, and past newsletter compositions.

## Architecture & Choices
*   **Agent Template:** `agentic_rag` (Retrieval-Augmented Generation agent with document search)
*   **Datastore:** `agent_platform_search` (Google Gemini Enterprise document search datastore for indexing and querying daily news articles)
*   **Deployment Target:** `agent_runtime` (Hosted fully-managed on Vertex AI Agent Runtime)
*   **CI/CD Pipeline:** GitHub Actions for multi-environment deployment and testing
*   **Version Control:** New local and remote Git repository
*   **Cross-Session Memory:** **Memory Bank** enabled to store user settings, preferences, and newsletter history

## Example Use Cases

### Scenario 1: Initial Profile Sync
*   **Input:** The user uploads or copy-pastes a CV specifying skills in *Python, Kubernetes, MLOps, and Large Language Models*.
*   **Process:** The agent parses the CV, extracts key concepts, and saves them in the Memory Bank.
*   **Output:** "I have analyzed your CV. I'll focus your newsletter on Kubernetes orchestration, MLOps pipeline tools, and Python AI developments. Would you like me to add any specific topics?"

### Scenario 2: Newsletter Generation
*   **Input:** User prompts: "Generate today's newsletter."
*   **Process:**
    1.  The agent retrieves the user's profile and preferences from the Memory Bank.
    2.  The agent queries the news corpus in `agent_platform_search` using terms like `MLOps GCP`, `Kubernetes v1.3x release`, `Python LLM frameworks`.
    3.  The agent filters search results, synthesizes summaries, and formats a markdown newsletter.
*   **Output:** A structured, personalized markdown newsletter with sections:
    *   **Major MLOps and LLM news today** (e.g., release of new Gemini models or Vertex pipelines)
    *   **Kubernetes updates** (relevant ecosystem changes)
    *   **Why this matters to your profile** (tailored explanation pointing to their specific CV skills)

## Tools Required

1.  **MemoryBankTool**: Built-in ADK tool to load, update, and search long-term memory records for user preferences.
2.  **DatastoreSearchTool**: Queries the indexed news corpus via the `agent_platform_search` service.
3.  **NewsSearchTool** (Optional/Secondary): A mock or live news API search tool to populate daily news into our ingestion pipeline.

---

## Constraints & Safety Rules
*   **Accuracy:** Never hallucinate news headlines or fabricate quotes. Every headline must be grounded in search results.
*   **Privacy:** The provided CV must be stored securely and treated confidentially.
*   **Relevance:** Strictly filter out news that does not intersect with the user's saved skill profile.

---

## Success Criteria
*   The agent accurately extracts skills from a raw CV and saves them to the Memory Bank.
*   The agent utilizes the retrieved skills to construct highly relevant queries for the `agent_platform_search` datastore.
*   The generated newsletter is well-formatted, clean, and directly relates search articles back to the user's experience.
*   Full CI/CD pipeline correctly builds, runs automated unit tests, and deploys the agent to Gemini Enterprise Agent Runtime on GCP.
*   Full observability is established with trace and log propagation to Cloud Logging and Cloud Trace.
